Binary Expression Tree

Introduction

我觉得二叉表达式树是最好的表达式树结构，因此我此后都使用二叉表达式树

以下是一个二叉表达式树的figure example

(Figure A. from wikipedia)

可以看到表达式树的叶子节点就是operand（运算元），非叶节点就是operator（运算符）

这里注意：表达式树的非叶子节点可以只有一个儿子，比如我们要表示$-x$的时候，就可以把减号$-$作为父亲节点，把operand $x$ 作为这个减号唯一的儿子节点

Construction of an expression tree

那么如何建构一个表达式树呢？

How to construct an expression tree?

Well. Let’s start from the simplest case. We use a postfix notation to construct the tree.

Use postfix notation

Description

For example, we got a postfix notation: a b + c d e + * *

First we prepare a stack to help us proceed construction.

Let’s iterate the notation with a loop.

So in the loop, if we encounter an operand, create a tree node to store it and push this node into our stack;

if we got an operator, pop the top two nodes in the stack. Use the two operands and one operator to construct a subtree. After that, push its root node into the stack.

Just proceed like this and we will finally build a beautiful tree out of the postfix notation.
