题目描述

有一条奶牛冲出了围栏，来到了一处圣地（对于奶牛来说），上面用牛语写着一段文字。

现用汉语翻译为：

有 N 个区间，每个区间[x,y]表示提供的y-x+1堆优质牧草。你可以选择任意区间但不能有重复的部分。

对于奶牛来说，自然是吃的越多越好，然而奶牛智商有限，现在请你帮助他。

输入格式

第一行一个整数 N。

接下来 N 行，每行两个数 x,y描述一个区间。

输出格式

输出最多能吃到的牧草堆数。

输入输出样例

输入 #1

Code

输出 #1

Code

说明/提示

$1≤n≤1.5×10^5,0\leq x\leq y\leq 3\times 10^6$

解

问题转化

转化问题时需要作出假设

假设目前我们已知区间$[0,z]$内牛能吃到的最多牧草堆数$f(z)$，这个时候我们可用$f(z)$和另一个已知的牧草堆区间$[x,y]$来得到$f(y)$ （其中$z\leq x$）。转移方程是$f(y)=max{f(z_i)+y_i-x_i+1}$

实现方法

用最暴力的方法做这个问题，我们每次都枚举当前的$y$（第一重循环），接着枚举我们所知的N个区间$[x_i,y_i]$（第二重循环），最后枚举$z$，其中$z\leq x_i$（第三重循环）

优化第三重循环

我们发现第三重循环的目标是求区间最大值，那么可以用树状数组/线段树把$O(z)$优化成$O(logz)$

优化第一、二重循环

发现有些$y$是冗余的，因为我们的N个区间是离散分布的，区间之间的那些点实际上不用枚举

这时候有两种处理方法：

  1. 数据坐标离散化：可以把$x_i,y_i$全部放到一个数组里排序然后枚举它们就好了，本来要枚举$3\times10^6$就下降到了$1\leq n\leq 1.5\times10^5$（第一重循环）。这个时候问题就变成了$f(z)=max{f(0),…,f(x_i-1)}+y_i-x_i+1$

    这个情况最终算法是动态规划+线段树维护

    干脆把第一、第二重循环合起来。我们预处理时先按左端点排序那N个区间，这样就可以用第一重循环遍历用于转移的区间，然后在循环内使用线段树找出可用于转移的过去值。

  2. 状态离散化：既然需要枚举的坐标是离散化的，而且是由已知的这些区间决定的，那干脆把状态改成$[1,N]$区间。原来的状态是已知一个右端点，通过新的更右边的区间转移，所以在更换状态的表达方式后我们仍旧要保留右端点的痕迹。这个时候可以考虑把已知区间按右端点排序。第一重循环枚举的是用于转移的区间，接下来用二分查找在已经在第一重循环遍历过的区间中找出合适的可用于转移的过去值，由于已经按右端点排序，我们知道二分查找作用的这个有序区间集合里，右端点和$f$值同时都是单调不下降的，所以二分查找能快速帮我们定位出最好的可用于转移的过去值。
