对象

定义

通称定义

对象是数据及基于这些数据的操作的集合

在computer中，一个对象实际上就是一片被分配的内存空间，这些内存可能是连续的，也可能是离散的，这片内存在更高层次上可作为一个整体来考虑，此整体就是一个对象。这片内存中存储着一系列数据以及可以对这些数据进行修改或读取的一系列代码。

Python视角

在Python中，对象就是为C中的结构体在堆上申请的一块内存

一般来说对象是不能被静态初始化的，也不能在栈空间上生存

唯一的是类型对象（如整数型对象，字符串型对象），Python中所有的内建的类型对象都是被静态初始化的

在Python中，一个对象一旦被创建，它在内存中的大小就是不变的了

但是有时候我们创建一个对象，希望它能封装一个可变长的数组或一个支持增加、删除节点的数据结构，怎么办呢？

这就意味着那些需要容纳可变长度数据的对象只能在对象内维护一个指向一块可变大小的内存区域的指针

  • Q: 但是为什么要设定“内存大小不变”这样的规则？

    A: 因为遵循这样的规则可以使通过指针维护对象的工作变得非常简单，一旦允许对象的大小可在运行期改变，我们就得考虑一下情形：在内存中有对象A，并且其后紧跟着对象B。如果运行期某时刻，A的大小增大了，这意味着必须将A整个移动到内存中的其他位置，否则A增大的部分就可能覆盖原本属于B的数据

    只要将A移动到内存中的其他位置，那么所有指向A的指针就必须立即得到更新，光是想想就知道这样的工作超繁琐

垃圾收集机制

Python 使用了引用计数这一简单技术来跟踪和回收垃圾。

在 Python 内部记录着所有使用中的对象各有多少引用。

一个内部跟踪变量，称为一个引用计数器(ob_refcnt)。

当对象被创建时， 就创建了一个引用计数， 当这个对象不再需要时， 也就是说， 这个对象的引用计数变为0 时， 它被垃圾回收。但是回收不是"立即"的， 由解释器在适当的时机，将垃圾对象占用的内存空间回收。

python

垃圾回收机制不仅针对引用计数为0的对象，同样也可以处理循环引用的情况。循环引用指的是，两个对象相互引用，但是没有其他变量引用他们。这种情况下，仅使用引用计数是不够的。

Python 的垃圾收集器实际上是一个引用计数器和一个循环垃圾收集器。作为引用计数的补充， 垃圾收集器也会留心被分配的总量很大（及未通过引用计数销毁的那些）的对象。 在这种情况下， 解释器会暂停下来， 试图清理所有未引用的循环。

在ob_refcnt之外，我们注意到ob_type是一个指向_typeobject结构体的指针，那么这个结构体是一个什么东西呢？

实际上该结构体对应Python内部的一种特殊对象，它是用来指定一个对象类型的对象，因此这里的ob_type可以简单看成类型信息。

因此，Python对象的核心是，一个引用计数，一个类型信息

而PyObject定义的内容仅仅是每一个Python对象都必须拥有的一部分内容，以我们将在下一章剖析的整数对象为例，对象中除了一上的核心内容外还有别的东西

c

Python的整数对象中，除了PyObject，还有一个额外的long变量，这个整数对象的实际数值就存在ob_ival中。同样，Python中的字符串对象、list对象、dict对象、成千上万的其他对象，都在PyObject之外保存了属于自己的特殊信息。
