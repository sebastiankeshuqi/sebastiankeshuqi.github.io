Intro

KMP: Knuth-Morris-Pratt

Main idea: make use of the pattern of the pattern string itself, look at the general picture

Example

Eg.1

Start from the beginning

Let A=”abababaababacb” B=”ababacb” (the index starts from 1).

Call A as the target string B as the pattern string.

Let’s start from the beginning. Set two pointers $p_a$ and $p_b$ on A[1] and B[1].

We identify that A[1:5]=B[1:5]=”ababa”. But A[6] != B[6].

So $p_a$ and $p_b$ set out from A[1] and B[1], but stopped on A[6] and B[6].

If we want to make use of the previous work, we have two strategies:

  • Shift $*p_a$ back to somewhere in A[1:5]
  • Shift $*p_b$ back to somewhere in B[1:5]

But reconsider the situation, pattern string is shorter:

  • for pre-processing
  • for cost in shifting (this one does not make sense actually)

So we’d better focus on pattern string.

How to shift

Manual observation

Look on A[1:5]. It’s “ababa”. Look on B[1:5]. It’s also “ababa”.

Manually observe the pattern. Shift $*p_b$ to B[3]. Now:

A[1:5]=”ababa”. B[1:3]=”aba”. $p_a\rightarrow A[5],p_b\rightarrow B[3]$. Now the matched substring is A[3:5]=B[1:3]=”aba”.

Essence

What is “aba” here?

It’s nothing but the suffix (A[3:5]) of A[1:5].

Now we got some rough feelings:
$$
A[1:5]=B[1:5]\quad A[3:5]=B[1:3]
$$
Because $B[1:3]=A[1:3]$ so $A[1:3]=B[1:3]=A[3:5]$, and $B[1:3]=B[3:5]$

Aha, when we are seeking a position for shifting we are actually searching for the equivalence between the prefix and suffix of B.

Therefore the problem is transformed into judging whether the prefix and suffix is the same.

Shift to where

We have other options when we shift the pointer.

We can also take $*p_b\rightarrow B[1]$. Here we have $A[5:5]=B[1:1]$. The matched substring here is shorter than “aba”.

We have to decide to settle with a longer or shorter substring? i.e. decide to choose “aba” or “a” here.

We can do “aba” because we have the second chance to go “a” in “aba”. (Here we are not sure about where to go so it’s better we move to the longest one and give a shot to every potential position)

Algorithm design

The design work has two parts:

  • Design the algorithm for shifting (pre-processing)
  • Design the algorithm for overall matching

Pre-processing

Data structure

B=”ababa”. Take p[j]: the len of the equal prefix and suffix before position j (exclusive).

For example, $B[1:p[j]] = B[j - p[j]:j-1]$

Work

Suppose we already pre-processed p[1] to p[j-1].

Transition one

If now $B[p[j-1]+1]==B[j]$, then $p[j]=p[j-1]+1$.

Say for example, j=4:

State j-1

State j-1: p[1]=1 p[2]=0 p[3]=1.

For j-1: in “aba”, $B[1] = B[3]$.

State j

For j: in “abab”, $B[1]=B[3]$ and $B[2]=B[4]$. So $p[j]=p[j-1]+1=2$

Transition two

If now $B[p[j-1]+1]!=B[j]$, then we shall consider if we can shorten the prefix to match current suffix.

Actually this process is similar to the matching process. Go to the matching part and go back to here when you finished reading it.

Ready ?

Go.

You imagine you are doing matching for two B strings.

Eg. 1

For B, p[1:] = [0,]

Matching

Set two pointers $p_a$ and $p_b$. Initially $p_a\rightarrow A[1],p_b\rightarrow B[1]$

If $p_a==p_b$, we just process $p_a++$ and $p_b++$

If $p_a!=p_b$, when $p_b\rightarrow B[j]$, we get $p_b\rightarrow B[p[j-1]]$

Shared complexity analysis

The loop occured in matching process is confusing.

But during the whole process, $p_b$ goes forward if and only if $p_b$ goes forward. So everytime $*p_b$ jumps back with array $p$, the time complexity is in control.

So the overall time is $T_n=cn$. Time complexity: $O(n)$.
